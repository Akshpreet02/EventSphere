	<h3
		class="c28"
		id="h.7qugy4517h09">
		<span class="c3">Purpose </span>
	</h3>
	<p class="c39">
		<span class="c12"
			>The purpose of this design document is to outline the development of our
			software &ldquo;EventSphere&rdquo;. It is an event management web
			application designed to simplify the planning, execution, and advertising.
			It also allows easy access for customers to be able to view all types of
			events occurring around them all within one application. It aims to
			address the common challenges in event management, such as collaboration,
			guest management, and promotion, by providing a comprehensive suite of
			tools that enhances these experiences for both event planners and
			attendees.</span
		>
	</p>
	<h3
		class="c28"
		id="h.3kzpbe3aglep">
		<span class="c3">Background Reading </span>
	</h3>
	<p class="c16">
		<span class="c12"
			>When first brainstorming about the project, the group had limited
			knowledge of the design. After consulting a few resources, a good
			understanding of what it means to have a successful event management web
			application was achieved. Documents found online gave insight into how the
			web application would operate [1] by showing important topics such as an
			attractive landing Page or homepage and Portfolio.</span
		>
	</p>
	<h3
		class="c28"
		id="h.vlvkst5wcbrp">
		<span class="c3">Context </span>
	</h3>
	<p class="c16">
		<span class="c5"
			>&quot;EventSphere&quot; is a web application designed to make event
			planning, managing, and finding events easier for everyone involved. Our
			app will help event organizers coordinate plans, manage guests, and
			promote their events more effectively. At the same time, it makes it easy
			for people to find and attend events that interest them. Our goal is to
			solve common problems in event organization like communication issues and
			guest management, making the whole process smoother for organizers and
			attendees alike.</span
		>
	</p>

		<span class="c3">Detailed Design </span>

	<ul class="c9 lst-kix_zg6d6jx3aobu-0 start">
		<li class="c10 li-bullet-0"><span class="c5">Features:</span></li>
	</ul>
	<p class="c32">
		<span class="c4"
			>The &quot;EventSphere&quot; app offers a range of features for easy event
			management, using a Client-Server model with MVC architecture. Users can
			create events, manage invitations and RSVPs, browse and sign up for
			events, get email updates, leave reviews, and share events on social
			media. It combines a React.js frontend with a Node.js backend using gRPC
			for communication, and JWT for security. The system supports CRUD
			operations on a MongoDB cloud database, ensuring a smooth flow of data
			from user actions to the database and back, making event management more
			efficient.11</span
		>
	</p>

	<h3
		class="c28"
		id="h.t74ddlifq07t">
		<span class="c3">Implementation Plan </span>
	</h3>
	<p class="c16">
		<span class="c5"
			>Our team has decided to split into 2 sub teams. One sub team will work
			primarily on the front-end and the other team will work primarily on the
			back-end. We have some individuals who will act as floaters and work on
			both teams as needed. Each team will plan and complete sprints that are
			broken down into manageable tasks to deliver features frequently, which
			will allow for any iterations that are needed to take place. We will have
			designated meetings/work time each week to ensure our team is making
			steady progress throughout the semester. We also hope that with weekly
			meetings team members will be held accountable to internal and external
			deadlines.</span
		>
	</p>
	<p class="c16"><span class="c5">&nbsp;</span></p>
	<p class="c16">
		<span class="c5"
			>The technologies that we intend to use will facilitate cloud
			based/web-based application programming, such as: React.js, Node.js,
			MongoDB (cloud instance on AWS), gRPC (Inter-process communication
			protocol).
		</span>
	</p>
	<p class="c16"><span class="c5">&nbsp;</span></p>
	<p class="c16">
		<span class="c5"
			>We intend to manage our source code via a Github repository that will be
			available to all team members. Every team member will implement the tasks
			that they are assigned in their own branch off the main repository to
			avoid merge conflicts.</span
		>
	</p>
	<p class="c16"><span class="c5">&nbsp;</span></p>
	<p class="c16">
		<span class="c5"
			>We will integrate the different front-end and back-end components
			implemented by different team members by having code review sessions
			before merging new functionalities into the main repository to mitigate
			git pull and push operations.</span
		>
	</p>
	<p class="c16"><span class="c5">&nbsp;</span></p>
	<p class="c16">
		<span class="c12"
			>We intend to do Exploratory testing once we finish implementing the
			functionality of a component to try and find most of the bugs, as soon as
			the component is developed. We also plan to do integrated testing once all
			the features are developed and combined in the final application.</span
		>
	</p>
	<p class="c26">
		<span class="c12"
			>Our deployment strategy is to use Vercel for our frontend in React.js and
			Render for our backend in Node.js.</span
		>
	</p>
